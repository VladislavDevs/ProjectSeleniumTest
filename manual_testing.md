# \# Ручные тест-кейсы для сервиса переводов F-Bank

# 

# \## Общая информация

# \- \*\*Сервис\*\*: F-Bank (фейковый банковский сервис)

# \- \*\*Базовый URL\*\*: `http://localhost:8000/?balance=10000\&reserved=3000` (параметры могут меняться)

# \- \*\*Предусловия для всех тестов\*\*: сервис запущен локально, открыта главная страница с заданными параметрами.

# 

# \---

# 

# \### Тест №1 (позитивный) – Успешный перевод при достаточном балансе

# 

# | Параметр | Значение |

# |----------|----------|

# | \*\*Название\*\* | Успешный перевод |

# | \*\*Предусловие\*\* | `balance=10000`, `reserved=3000` (доступно 7000) |

# | \*\*Шаги\*\* | 1. Нажать на карточку \*\*«Рубли»\*\*.<br>2. В поле номера карты ввести 16 цифр: `1234567812345678`.<br>3. В поле суммы ввести `5000`. |

# | \*\*Ожидаемый результат\*\* | Появляется кнопка \*\*«Перевести»\*\*. При её нажатии появляется alert с текстом: «Перевод 5000 ₽ на карту ... принят банком!». |

# | \*\*Фактический результат\*\* | (заполняется при тестировании) |

# | \*\*Статус\*\* | Pass / Fail |

#

# \### Тест №2 (негативный) – Превышение доступной суммы

# 

# | Параметр | Значение |

# |----------|----------|

# | \*\*Название\*\* | Блокировка перевода при недостатке средств |

# | \*\*Предусловие\*\* | `balance=10000`, `reserved=3000` (доступно 7000) |

# | \*\*Шаги\*\* | 1. Нажать на карточку \*\*«Рубли»\*\*.<br>2. Ввести номер карты (16 цифр).<br>3. В поле суммы ввести `8000`. |

# | \*\*Ожидаемый результат\*\* | Кнопка \*\*«Перевести»\*\* не появляется. Вместо неё отображается сообщение: \*\*«Недостаточно средств на счете»\*\*. |

# | \*\*Фактический результат\*\* | (заполняется при тестировании) |

# | \*\*Статус\*\* | Pass / Fail |

#

# \### Тест №3 (граничный) – Округление комиссии вниз

# 

# | Параметр | Значение |

# |----------|----------|

# | \*\*Название\*\* | Проверка округления комиссии (10% вниз) |

# | \*\*Предусловие\*\* | `balance=1000`, `reserved=0` (доступно 1000) |

# | \*\*Шаги\*\* | 1. Нажать на карточку \*\*«Рубли»\*\*.<br>2. Ввести номер карты (16 цифр).<br>3. В поле суммы ввести `909`. |

# | \*\*Ожидаемый результат\*\* | Комиссия = `floor(909 \* 0.1) = 90`. Итог = 999 ≤ 1000. Кнопка \*\*«Перевести»\*\* появляется. |

# | \*\*Фактический результат\*\* | (заполняется при тестировании) |

# | \*\*Статус\*\* | Pass / Fail |

#

# \### Тест №4 (негативный) – Номер карты не 16 цифр

# 

# | Параметр | Значение |

# |----------|----------|

# | \*\*Название\*\* | Валидация длины номера карты |

# | \*\*Предусловие\*\* | Любое состояние счёта |

# | \*\*Фактический результат\*\* | (заполняется при тестировании) |

# | \*\*Статус\*\* | Pass / Fail |

# \### Тест №5 (негативный) – Ввод букв в поле суммы

# 

# | Параметр | Значение |

# |----------|----------|

# | \*\*Название\*\* | Блокировка нечисловых символов в сумме |

# | \*\*Предусловие\*\* | Номер карты уже введён (16 цифр), поле суммы активно. |

# | \*\*Ожидаемый результат\*\* | Поле \*\*не должно принимать\*\* буквы (либо они игнорируются, либо появляется ошибка «Некорректная сумма»). |

# | \*\*Фактический результат\*\* | (заполняется при тестировании) |
